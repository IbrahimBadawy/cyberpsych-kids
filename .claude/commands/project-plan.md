# تخطيط المشروع - Project Plan

## متى تستخدم هذه المهارة
- تحويل مشروع من مرحلة المناقشة إلى التخطيط
- كتابة خطة تفصيلية مع جدول زمني
- تحديد الموارد والأدوات المطلوبة

## المبدأ الأساسي
**الخطة الجيدة تجيب على: ماذا؟ لمن؟ كيف؟ بماذا؟ متى؟ - قبل ما تبدأ حرف واحد من التنفيذ.**

## المدخلات
- اسم المشروع (project) - مطلوب

## الخطوات
1. **اقرأ** كل ملفات `01-discussion/` لفهم نتائج المناقشة
2. **اقرأ** `status.json` وحدّث المرحلة إلى `planning`
3. **اكتب** الخطة التفصيلية في `02-plan/plan.md`:

```markdown
# خطة مشروع: [الاسم]

## 1. نظرة عامة
- الهدف النهائي
- الجمهور المستهدف
- المخرجات المتوقعة

## 2. المحتوى المطلوب
| # | العنصر | النوع | الحالة |
|---|--------|-------|--------|
| 1 | [عنوان] | [قصة/فيديو/لعبة] | [ ] |
| 2 | ... | ... | [ ] |

## 3. الشخصيات
- [قائمة الشخصيات المطلوبة مع مرجع character-bible]

## 4. الأدوات المستخدمة
| الأداة | الاستخدام | مجاني/مدفوع |
|--------|----------|-------------|
| Claude | كتابة المحتوى | مدفوع |
| ... | ... | ... |

## 5. الجدول الزمني
| المرحلة | المدة | البداية | النهاية |
|---------|-------|---------|---------|
| تصميم الشخصيات | ... | ... | ... |
| كتابة المحتوى | ... | ... | ... |
| التصميم البصري | ... | ... | ... |
| المراجعة | ... | ... | ... |
| النشر | ... | ... | ... |

## 6. معايير النجاح
- [ ] ...
```

4. **اكتب** `02-plan/timeline.md` بالتفصيل
5. **اكتب** `02-plan/resources.md` بالأدوات والمواد
6. **اعرض** الخطة على المستخدم للموافقة
7. **عند الموافقة:** اقترح الانتقال للتنفيذ (`/project-start`)

## أمثلة
```
/project-plan "سلسلة-قصص-التنمر"
/project-plan "دورة-الأمان-للمعلمين"
```
