# بدء التنفيذ - Start Implementation

## متى تستخدم هذه المهارة
- الانتقال من مرحلة التخطيط إلى التنفيذ الفعلي
- إنتاج المحتوى حسب الخطة المعتمدة
- تتبع التقدم وتحديث الحالة

## المبدأ الأساسي
**التنفيذ يتبع الخطة. كل محتوى يُنتج يُحفظ في فولدر المشروع مع تتبع التقدم.**

## المدخلات
- اسم المشروع (project) - مطلوب
- --item [رقم العنصر من الخطة] - اختياري (لتنفيذ عنصر محدد)

## الخطوات
1. **اقرأ** `projects/[المشروع]/02-plan/plan.md` للخطة
2. **اقرأ** `projects/[المشروع]/03-implementation/progress.md` للتقدم الحالي
3. **حدّث** `status.json` → `"phase": "implementation"`
4. **نفّذ** العنصر التالي (أو المحدد بـ --item):
   - استخدم الأمر المناسب (/story, /video-script, /game-design, إلخ)
   - احفظ المخرج في `03-implementation/`
   - أنشئ فولدر فرعي لكل عنصر إذا لزم
5. **حدّث** `progress.md`:

```markdown
# تتبع التقدم

| # | العنصر | الحالة | تاريخ البدء | تاريخ الانتهاء | الملف |
|---|--------|--------|-------------|----------------|-------|
| 1 | [عنوان] | ✅ مكتمل | 2026-04-13 | 2026-04-13 | [رابط] |
| 2 | [عنوان] | 🔄 جاري | 2026-04-14 | - | [رابط] |
| 3 | [عنوان] | ⏳ قادم | - | - | - |

## سجل التنفيذ

### [التاريخ] - [العنصر]
- ما تم إنتاجه
- ملاحظات
- الخطوة التالية
```

6. **بعد اكتمال كل عنصر:** اقترح تشغيل `/review` عليه
7. **بعد اكتمال كل العناصر:** اقترح `/project-status` للمراجعة النهائية

## أمثلة
```
/project-start "سلسلة-قصص-التنمر"
/project-start "سلسلة-قصص-التنمر" --item 3
```
